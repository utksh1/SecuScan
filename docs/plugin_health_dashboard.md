# Plugin Health & Coverage Dashboard

The Plugin Health & Coverage Dashboard is a developer utility for inspecting SecuScan plugins.

It follows the repository plugin model by treating plugin directories containing `metadata.json` as the source of truth and checking whether each plugin has a corresponding `parser.py`.

## What it checks

- Total plugin count
- Parser availability
- Plugin category distribution
- Plugin metadata path
- Plugin directory path

## Step-by-Step Walkthrough

### 1. Running the Dashboard

Start by generating a report. For a quick terminal view:

```bash
python scripts/plugin_health_dashboard.py
```

For structured data (useful for automation or CI checks):

```bash
python scripts/plugin_health_dashboard.py --format json --output health.json
```

### 2. Reading the Output

The Markdown output presents findings in this order:

1. **Summary block** — total plugins found, how many have parsers, category breakdown
2. **Healthy plugins** — plugins that have both `metadata.json` and `parser.py`
3. **Missing parsers** — plugins that have `metadata.json` but no `parser.py` (these should be investigated)
4. **Category summary** — count of plugins per category (crawler, exploit, scanner, etc.)

### 3. Interpreting Health Indicators

| Indicator | Meaning | Recommended Action |
|-----------|---------|-------------------|
| Has `metadata.json` + `parser.py` | Healthy — plugin is fully wired | None |
| Has `metadata.json`, no `parser.py` | Incomplete — results cannot be parsed | Add `parser.py` or flag for removal (check `plugins_without_parser` in JSON summary) |
| `metadata.json` missing | Unregistered — plugin not discoverable | Add `metadata.json` or remove plugin directory |

### 4. Taking Action on Unhealthy Plugins

For plugins missing a parser:
1. Review the plugin source in `plugins/<plugin-name>/`
2. If the plugin is active, implement `parser.py` following the existing pattern in `plugins/template/`
3. If the plugin is deprecated, remove its directory or add it to the ignore list in `scripts/plugin_health_dashboard.py`
4. Re-run the dashboard to verify the fix

### 5. Automating in CI

The JSON output is suitable for CI validation. Example check:

```bash
python scripts/plugin_health_dashboard.py --format json | \
  python -c "import json,sys; d=json.load(sys.stdin); exit(1 if d['summary']['plugins_without_parser'] > 0 else 0)"
```

This exits with code 1 if any plugins are missing parsers, making it suitable as a CI gate.

## How to run

Print a Markdown report to the terminal:

```bash
python scripts/plugin_health_dashboard.py
```

Print JSON output:

```bash
python scripts/plugin_health_dashboard.py --format json
```

Write Markdown output to a file:

```bash
python scripts/plugin_health_dashboard.py --output plugin_health_report.md
```

Write JSON output to a file:

```bash
python scripts/plugin_health_dashboard.py --format json --output plugin_health_report.json
```

## Notes

The script does not write generated reports by default. Reports are printed to stdout unless an explicit `--output` path is provided.
